Overview

Each overseas subsidiary had been operating its own VoC system and CI/CD environment independently. I participated in planning a unified platform that standardizes them all under HQ standards, serving as Technical Architect (TA) throughout.

Roles up to the first pilot release

  • Led the overall infrastructure & software architecture design for both development and production servers.
  • Addressed closed-network constraints — configured Red Hat 9.4 RPM packages and resolved OS-version compatibility issues.
  • Designed dev-environment workarounds for mismatches with the authentication server, and established a migration plan for production.
  • Provided developers with a fully equipped dev environment and delivered technical guidelines.
  • Performed end-to-end TA duties — technical communication with the client, requirements analysis, and deployment-strategy design.

개요

1차 파일럿 릴리스까지의 역할

  • 개발·운영 서버 양쪽의 전체 인프라 & 소프트웨어 아키텍처 설계 주도.

  • 인증 서버 불일치에 대한 개발 환경 우회책 설계 및 운영 배포를 위한 마이그레이션 계획 수립.
  • 개발자들에게 완비된 개발 환경 제공 및 기술 가이드라인 전달.
  • 고객 기술 커뮤니케이션, 요구사항 분석, 배포 전략 설계 등 end-to-end TA 업무 수행.
